Distribution of ordinal scaled target values


  • The distribution of the target variable, "Ordinal education label", is still imbalanced after conversion from categorical descriptors to the 1-4 ordinal system.

  • Choice of machine learning method used to create the Education Level Predictor is influenced by this imbalance. Whatever training method is chosen, it must be able to handle the error produced from the imbalance. Ensemble methods such as Random Forest Classifiers and AdaBoost may be usefull in this context.

Alternative 1-3 ordinal scale

  • In case it proves to be too difficult for our model to distinguish the underrepresented target variable classes (specifically, '1', '2', and '3') an alternative target labelling system will be created for comparisson. New labels will be stored as new column 'Education level 3'
  • Education levels classes 1,2,3 will become aggregated as class 1. Classes 4 and 5 will be relabelled to be classes 2 and 3, respectively.

NLP based features - rationale:

  • The following feature set will be engineered from the post-html-parsing, pre-lemmatization essay answers:
    • number of characters
    • number of words total
    • number of dictionary-words
    • number of punctuation

Before starting this section of the project, we need to consider a question:


  • Is better to consider each of the essay question answers seperately (ie create NLP engineered feature columns and, later, Tf-IDF vectors for each answer a user provided)...

    or

  • ...is it more informative to our model to consider the entire corpus of what the user ha written (ie create one set of NLP engineered feature columns and train Tf-IDF on the entire corpus of the 9 essay answer columns, producing one single sparse matrix for each user?

Two Options:


  1. Consider Each Essay Answer Separately:
    • Advantages:
      • Captures the unique characteristics and vocabulary of each essay answer individually.
      • Allows the model to distinguish between the importance of terms within each essay answer.
    • Considerations:
      • Each essay answer is treated as an independent document, which may not fully capture relationships or patterns across multiple answers from the same user.
      • Increases the dimensionality of the feature space, which may lead to higher computational costs, especially if the dataset is large.

  1. Consider All Essay Answers Together:
    • Advantages:
      • Creates a single TF-IDF and NLP feature set representation for each user, incorporating information from all essay answers.
      • May capture overall patterns or themes across multiple answers, providing a holistic view of the user's profile.
    • Considerations:
      • Assumes that all essay answers contribute equally to the user's profile, which may not always be the case.
      • The combined text may dilute the importance of individual answers, especially if some answers are more relevant to the prediction task than others.

Why not do both?

We will save the output from the two methods in feature columns of out training dataframe, to be loaded and experimented with during the model training section of the project.

    # Calculate TF-IDF scores

    # Normalize TF-IDF matrix

    # Initialize and train Word2Vec model
    word2vec_model = Word2Vec(sentences=[doc.split() for doc in text_data], vector_size=100, window=5, min_count=1, sg=1)

    # Obtain word embeddings for each document weighted by TF-IDF scores
    word_embeddings = []
    for i, doc in enumerate(text_data):
        doc_words = doc.split()
        doc_embedding = []
        for word in doc_words:
            if word in word2vec_model.wv and word in tfidf_vectorizer.vocabulary_:
