Introduction:¶


  • This project analyzes data from the on-line dating app OKCupid.


  • With the nearly ubiquitous use of such apps on the modern-day dating scene, it becomes an interesting question how these apps recommend people to eachother. Understanding this process better can give profound insight into a foundational feature of our society.


  • Data source: The file 'profiles.csv' was provided by Codecademy.com


Scoping:¶

Project Goals:¶

This project seeks to answer two research questions:

  1. Can a person's education level be predicted based on their written responses to essay questions?

  2. Can app users' data be leveraged to create a recommender system that suggests potential romantic matches, based on their profile?

Data:¶

  • The project has one data set provided by Codecademy called 'profiles.csv'.
  • Each row represents an OKCupid user.
  • Each column represents answers provided by that user to various multiple choice and short answer (essay) questions.

Analysis:¶


  • First, descriptive statistics and data visualization will be used to explore the distribution, count, and relationship between variables.


  • Next, natural language processing (NLP) and supervised classification algorithms will be implemented to create a model ("Education Level Estimator" or "ELE")that will predict a user's education level based on their answers to the essay questions they submitted.


  • Finally, unsupervised clustering algorithms will be harnessed to create a recommender system ("Match Recommender" or "MR"), matching potential romantic partners based on their location in high-dimensional space.


Evaluation:¶


  • The project will conclude with the evaluation of both models.


  • The Education Level Estimator ("ELE") will be evaluated using a confusion matrix, as well as accuracy, precision, recall, and F1 scores (recall: ELE is a supervised learning model, with known, true values).


  • The Match Recommender ("MR") will be evaluated qualitatively be exploring the centroids of the clusters of profile traits produced by the model. An attempt will be made to succinctly name each of the profile types produced by the model.

Remove outliers in the continuous variables - Age, Height, and Income:¶

  • Very long tails suggest we are dealing with extreme outliers in all three columns.
  • Outliers in the height and age columns can be dismissed as impossible or close enough to merit extreme skepticisim, and will be dropped (for example, ages over 100, heights less than 48" or greater than 84", etc.).
  • However, outliers in the income column require more careful examination. A high value income can be very indicative of a person's education level and these outliers may not represent "bad" data (ie. if the user has provided true answers to the questionnaire and happens to make a large income, they could inform our model and should be kept). Scaling the continuous variables will minimize the impact of the large integer values in this column on the overall training of the model.
  • We will drop the age and height column now. Once our target variable ('education') has been engineered, we will return to the idea of dropping certain rows where income is listed as 500,000 or 1,000,000 dollars. 
#Define function that will remove continuous variables that are beyond a certain threshold
#number of standard deviations above or below the mean:
def remove_outliers_std(df, column_name, threshold=3):
    
    mean = df[column_name].mean()
    std_dev = df[column_name].std()
    lower_bound = mean - threshold * std_dev
    upper_bound = mean + threshold * std_dev
    
    # Filter out rows with values outside the lower and upper bounds
    df_filtered = df[(df[column_name] >= lower_bound) & (df[column_name] <= upper_bound)]
    
    # Drop rows with NaN values
    df_filtered = df_filtered.dropna(subset=[column_name])
    
    return df_filtered

Summary¶


  • The project's research questions were established.


  • Analysis methods and Evaluation criteria were proposed and outlined.



  • Data was preprocessed in preparation for the Feature Engineering stage of the project, as follows:
    • HTML parsing of essay answers and offspring column, using Beautiful Soup library.
    • Removal of stop words and Lematization of essay answers using WordNetLemmatizer.
    • 'location', 'speaks', and 'ethnicity' columns processed to contain lists, ready for one hot encoding

  • Missing values in the 'income' variable were replaced using mean imputation. These missing values may not have been randomly missing given the context (i.e. people on dating apps have varying reasons for reporting or not reporting their incomes).


  • In the next steps of the project, it will be necessary to test if the difference between the distribution of missing incomes across the target classes is statistically different from the general popoluation's distribution across the target classes. If there is no significant difference, income could be kept as a scaled variable. Furthermore, it can be assumed that PCA in future stages of the project will handle any potential biases introduced by mean imputation.
